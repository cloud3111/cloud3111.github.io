序言

​ 在写这个心得之前，我内心充满了复杂的情绪，因为写心得对我这个散漫成性的人来说，无疑是一项重大的挑战，哈哈。 关于这个项目：“前端通用系统”，听起来颇具吸引力，但细究下来会发现，它缺乏明确的目的性和定位。我是一名个人练习生 不不不 串台了 我是一名主学后端的开发者 , 虽然系统学习前端项目对我来说并不是那么重要，但出于对前端的兴趣和对知识的好奇心，我还是想试试看。 说白了，就是有一天我突然产生了想学习前端的念头，这个念头一经冒出，便引起了世界的广泛关注，大型纪录片持续为您播出……哈哈，又偏题了… 不说那么多废话了，我们直接分析这个项目吧。(嗯哼嗯哼!! 我是小白别喷我 即使我有多菜鸟)

初识

  • 技术栈: Vue3 + Vite + Axios +ElementPlus + Mock + Pinia + Router

  • 一个很典型的前端框架, 但对于我来说只能说道阻且长 毕竟我还是0开发哦

  • 想要了解这个项目 首先得了解Vue3的特性: 自定义组件, 库函数和挂载

  • vue项目都会有一个入口main.js文件 => 其中一个作用就是创建一个主组件实例 将这个实例挂载到DOM元素上(也就是最原始的html元素) =< 还有一个作用是创建或者导入各式各样的插件供实例使用

  • 注意组件的生命周期: OnMouted 每次启动都会渲染一次

路由

  • 其次我认为很重要的是路由 他的作用是: 支持页面跳转 添加路由组 子路由 可以添加路由规则搭建 守卫路由

  1. 配置路由组(路由表)
  1. 跳转实际方法
  1. 在视图组件汇总配置router-view标签

Pinia

拥有组合式 API 的 Vue 状态管理库, 它允许你跨组件或页面共享状态

可以在多个组件之间共享数据，并且共享的数据是响应式的，即数据的变更能及时渲染到模板

pinia的核心就是 store（仓库）“store”基本上就是一个容器方法对象

state代表属性对象 actions代表方法集 getter代表public访问

  • 在main.js中定义Pinia实例
  • 在src/store/index.js文件中集中定义和管理共享数据
  • 在组件中使用

ElementPlus

  • 然后是ElementPlus组件库, 我第一次接触到时候真的被惊艳到了,不是吧哥们?? 这都能直接调用吗…但是它的复杂程度也是独一档 什么属性 事件 插槽啊 真的是看得我眼花缭乱的

  • 布局也依照elementPlus规范 总体组件采用 container 布局容器

  • 组件的二次排版可以采用layout布局,通过 row 和 col 组件，并通过 col 组件的 span 属性我们就可以自由地组合布局。

表单

  • 表单: 主要是v-bind绑定一个formObject表单对象 每一个表单项都映射一个表单对象字段(v-model绑定)
  • 属性绑定和双向绑定: input用双向可以随时进行修改不会出现数据死绑

表格

  • 表格: 和表单不一样的是表格是用列prop渲染的

菜单

  • 剩下的要讲的重要函数是v-for渲染,将结合menu菜单组件来讲

  • :key 是用来为每个循环渲染的对象提供一个唯一的标识 确保每个嵌套的 v-for 都有自己的 key, 解决避免因为深度嵌套而导致的问题

  • :index 是当前项的索引,但它可以用作循环中的一个变量，表示当前项在数组中的位置。

API请求

对js部分也就是都可以加上setup语法糖来对外暴露 减少代码量

对css部分就是要加上 scope 和 lang=”less” 避免丢失html语法标签如:p h

  • 这部分讲的是API请求,这是前后端联调中最重要的一环: 能 发起Http请求有 axios 和 fetch 和 xmlHttp
  • axios需要导入类库,内嵌拦截器, 而fetch是浏览器自带的,不依托额外库,本身语法简单,但是没有拦截器等等
  • async await 成对出现 异步处理函数 === !同步
  • params：用于 GET 请求的查询参数（即 URL 中的查询字符串部分）。通常是 URL 中传递的参数，表示附加在请求 URL 后的键值对。例如：/api/resource?key=value。
  • data：用于 POST 或其他请求方法（如 PUT、PATCH）的请求体数据。在 POST 请求中，数据通常作为请求的正文发送。

总结

成为一位卓越的前端工程师，关键在于项目启动之初的精心规划和布局。就像建筑大师在动工前绘制蓝图，像诸葛亮叫阵前的排兵布阵一般, 我们也需要先搭建起项目的框架，再逐步填充内容。此外，处理数据流——从发送请求到接收和渲染数据——无疑是技术挑战的核心。这里没有捷径可走，唯有通过不断的实践、探索和动手操作，才能掌握其中的精髓。记住，每一次的尝试都是向前迈进的一步，勤于实践，终将成就卓越。”
