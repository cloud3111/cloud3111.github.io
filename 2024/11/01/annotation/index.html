1.什么是注解

  • 注解和注释不同,当编译器运行时,注释会被跳过,
  • 而注解不会,反而会反过来影响程序 如:Tomcat, 编译器, 框架
  • 通过直接描述代码的行为来告诉框架或编译器应该做什么
  • 注解通常与反射机制结合使用，通过反射来在运行时获取注解信息，从而根据注解内容执行特定逻辑

2.注解的作用

  • 标识 类: 当程序启动时,扫描所有类的时候, 标识当前类为… 例如:@WebServlet用来标识当前类为Servlet, 并管理类的生命周期(启动,运行,销毁)
  • 赋予 类: 标识类时赋予类的作用时间(RUNTIME) Spring 使用注解来自动配置 Bean(使bean在程序执行运行时return唯一实例)
  • 调用 类: 本身不做实现, 利用反射, 比如加了一个运行时注解, 那这个程序就会在运行时被执行, 而不用去创建实例对象.方法
  • 标识方法的过程解析: 遍历所有方法 找到被标识的方法 反射执行

3.注解的组成

指定两个元注解:@Target and @Retention

@Target 指定注解的作用对象

  • @ElementType :

    • ElementType.TYPE 类 接口

      • ElementType.METHOD 方法(成员方法)

      • ElementType.FIELD 属性(成员变量)

      • ElementTYPE.CONSTRCTRUE 构造器

      • ElementTYPE.PACKAGE 包

      • ElementTYPE.ANNOTATION 注解

  • @RetentionPolicy: 指定注解的作用域
    • RetentionPolicy.SOURCE 源代码级别，由编译器处理，处理之后就不再保留
    • RetentionPolicy.CLASS 注解信息保留到类对应的字节码文件中
    • RetentionPolicy.RUNTIME 由JVM(解释器)读取，运行时使用

5.补充

  • AnnotatedElement 接口是 Java 反射 API 中的一个关键接口，用于操作和读取注解。它由 Class、Method、Field 等类实现，允许在运行时检查注解信息。AnnotatedElement 的接口方法使得我们可以在运行时灵活地检查并获取注解的详细信息，从而决定如何动态处理类、方法或字段。

  • Spring 就大量使用了这一模式：扫描注解来注入依赖、管理事务、控制请求等。

  • // 也是利用反射
Class clazz = testAnnotation.class;
Annotation annotation = clazz.getAnnotation(InitMethod.class);
if (deprecatedAnnotation != null) {

Annotation[] annotations = clazz.getAnnotations();
for (Annotation annotation : annotations) {
    System.out.println("注解类型: " + annotation.annotationType().getName());
