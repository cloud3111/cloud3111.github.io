Git 常用命令总结

Git 基础概念

  • Pull Request：GitHub 的合并请求，拉取请求是针对 main 分支的。
  • Git 区域：
    • 工作区 → 暂存区 → 本地仓库 → 远程仓库

  • untracked 文件：在本地仓库中存在, 但是没有被git add加入暂存区, 而是被放在工作区; 但是在远程仓库中有同名文件, 这时候不知道该保留本地 untracked 的版本，还是以合并进来的版本为准
  • unstaged 文件：staged状态是文件被git add加入暂存区, 而unstaged状态是在工作区又修改了文件, 这时在暂存区的文件就显示unstaged
  • git stash: 会把所有未提交的修改（包括暂存的和非暂存的）都保存起来，用于后续恢复当前工作目录。可以通过 git stash pop命令恢复之前缓存的工作目录

Git进阶命令

Git Stash 临时存储

  • 把当前工作区和暂存区的改动临时保存起来，然后让工作区恢复到干净的状态（和最近一次提交一致）-> 临时本地保存
  • 案例

Git Rebase 分支基变

  • 基变与合并的区别就是 将提交记录变成直线, 让历史更清晰简洁
  • 参考文献

    Git Reset 恢复回滚

  • HEAD 是 Git 中的一个指针，它指向当前分支上的最新一次提交
  • 重置当前分支的 HEAD 位置,
命令 暂存区 暂存区 描述
git reset —soft 不变 不变 只移动HEAD
git reset —mixed（默认） 清空 不变 改变HEAD + 暂存区
git reset —hard 清空 清空 改变HEAD + 暂存区 + 工作区

Git Feach 拉取不合并

  • git fetch 只是把远程仓库的更新下载到了本地 .git 目录中，不会直接影响本地的工作区和当前分支, 需要你手动merger合并或者rebase基变
  • git pull则是会获取所有远程索引并合并到本地分支

    Git Reflog 本机历史记录

  • 记录了本地仓库中分支的变化记录 和 HEAD的移动历史, reflog 是本地记录，远程仓库没有。
  • 包括了- 提交（git commit）合并（git merge）变基（git rebase）撤销（git reset）拉取（git pull）checkout（切换分支）甚至删除分支也会有记录

    Git 标签管理

Git问答

  • 问: 使用 git merge 的时候，Git 报错提示有一些本地的未跟踪（untracked）文件，而这些文件在合并进来的分支中也存在，导致无法完成合并, 如何解决?

  • 答: 使用git clean对不重要的本地untracked文件进行删除, 或者先把文件复制到其他地方备份, 或者通过git stash 进行保存, 或者通过git add 把文件交给版本管理

  • 问: 如何忽略本地的某些文件? .gitignore 不生效?

  • 答: 只有未被提交的文件才能真正被git忽略, 否则只要提交过一次就有了这个文件的追踪记录, 如果想真正让 Git 忽略已经被跟踪的 log/ 文件夹，需要先把它们从 Git 索引中移除，然后再让 .gitignore 生效

Tips

  • 在 Git 的操作设计中，任何涉及可能覆盖工作区内容的行为，都会默认中止或警告，防止意外的数据丢失。这是 Git 保守而安全的设计哲学。
